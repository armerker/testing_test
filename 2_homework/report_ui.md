## 1. Введение

Целью данной работы является тестирование требований к функции поиска на Яндекс.Диске, а также проверка реализации данной функции на соответствие этим требованиям. Скриншоты результатов находятся в папке `screen_ui`

---

## 2. Тестирование требований (Requirements Testing)

В данном разделе проведен анализ исходных требований к функции поиска на соответствие критериям качества из статьи.

**Исходный текст требований:**
> ○ Реализована функция поиска по файлам и папкам.
> 
> ○ Поле поиска принимает любые буквы/цифры/символы.
> 
> ○ Результат работы функции - набор файлов и папок, удовлетворяющих запросу.
> 
> ○ В случае, если нет файлов, удовлетворяющих запросу, получено сообщение "В вашем Диске не найдено файлов или папок по запросу <.......>"
> 
> ○ Если файл находится в корзине - он так же может быть найден поиском.

### 2.1. Анализ на основе критериев качества

| Критерий | Оценка | Комментарий |
|:---------|:-------|:------------|
| **Полнота** | **Не полностью** | Требования не описывают поведение системы при вводе спецсимволов (`*`, `?`, `#`), пробелов или очень длинных запросов. Также не указан порядок сортировки результатов. |
| **Ясность** | **Неясно** | Пункт «любые символы» слишком размыт. Пункт про корзину не уточняет, *как именно* файл должен быть найден (с пометкой "в корзине" или как обычный файл). |
| **Тестируемость** | **Тестируемо** | Несмотря на неточности, по этим пунктам можно написать тесты (позитивные/негативные сценарии). |
| **Согласованность** | **Согласовано** | Противоречий между пунктами не обнаружено. |
| **Корректность** | **Корректно** (в общем) | Требования описывают ожидаемую логику работы поиска. |

### 2.2. Вывод по тестированию требований

Требования требуют доработки: необходимо уточнить поведение системы с различными типами входных данных и добавить критерии отображения результатов (сортировка, индикация файлов в корзине).

## 4. Баг-репорты (Bug Reports)

На основе обнаруженных несоответствий были заведены следующие баг-репорты.

### Баг-репорт №1: Несоответствие текста сообщения об ошибке

*   **Заголовок:** Текст сообщения при отсутствии результатов поиска не соответствует спецификации.
*   **Проект:** Яндекс.Диск (Web-версия)
*   **Серьезность (Severity):** Minor (Низкая. Текст не соответствует документации, но функция информирования пользователя работает).
*   **Приоритет (Priority):** Low (Можно исправить в следующем релизе).
*   **Окружение:** ПК, Windows 11, Браузер Chrome.
*   **Шаги для воспроизведения:**
    1.  Открыть Яндекс.Диск.
    2.  Ввести в поиск заведомо несуществующее значение (например, `ghbdtn`).
    3.  Нажать Enter.
*   **Фактический результат:** Отображается сообщение «Ничего не нашлось».
    *(Приложение: `файл находится в корзине.png`)*
*   **Ожидаемый результат:** Должно отображаться сообщение «В вашем Диске не найдено файлов или папок по запросу <...>», согласно требованию.

### Баг-репорт №2: Общий поиск не индексирует файлы в корзине

*   **Заголовок:** Общий поиск по Диску не находит файлы, находящиеся в корзине.
*   **Проект:** Яндекс.Диск (Web-версия)
*   **Серьезность (Severity):** Major (Значительная. Нарушено заявленное требование, функционал работает не полностью).
*   **Приоритет (Priority):** High (Высокий. Требуется исправление в ближайшее время).
*   **Предусловия:** На диске есть файл, перемещенный в корзину.
*   **Шаги для воспроизведения:**
    1.  Открыть Яндекс.Диск.
    2.  Ввести в общий поиск название файла из корзины.
    3.  Нажать Enter.
*   **Фактический результат:** Система не находит файл и отображает экран «Ничего не нашлось», предлагая искать в корзине отдельно.
    *(Приложение: `файл находится в корзине.png`)*
*   **Ожидаемый результат:** Файл должен быть найден и отображен в результатах общего поиска (возможно, с серой иконкой или подписью «в корзине»), согласно требованию.

## 5. Общий вывод

В ходе работы были протестированы требования и реализованная функция поиска.

1.  **Требования** содержат неточности и нуждаются в уточнении (полнота и ясность).
2.  **Система** частично соответствует требованиям. Основная функция поиска работает, но обнаружено два дефекта:
    *   Неверный текст сообщения об отсутствии результатов.
    *   Некорректная работа поиска с файлами в корзине (общий поиск их не видит).

Дефекты оформлены в виде баг-репортов и требуют исправления.
