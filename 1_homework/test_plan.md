# Тест-план: GET /products/{productId}/status

**Версия:** 1.0  
**Дата:** 06.03.2026  
**Автор:** Пугачев Андрей  

---

## 1. Объект тестирования

REST API эндпоинт `GET /products/{productId}/status`.  
Параметры запроса:

- `authToken` (обязательный) — 16 символов, буквы + цифры
- `recalculate` (опциональный)
- `owner` (опциональный)
- `region` (опциональный)

Параметры `recalculate`, `owner` и `region` **не коррелируют** между собой.

---

## 2. Что тестируем

Проверке подлежат следующие функциональные области:

- Авторизация и аутентификация с использованием параметра `authToken`
- Обработка параметра `recalculate`
- Обработка параметра `owner`
- Обработка параметра `region`
- Комбинации независимых параметров `recalculate`, `owner` и `region`
- Обработка идентификатора продукта `productId`

---

## 3. Как будем тестировать

При планировании тестирования используются следующие техники тест-дизайна:

- **Классы эквивалентности и граничные значения** — для проверки длины и формата `authToken` (15, 16, 17 символов, только буквы, только цифры, спецсимволы)
- **Попарное тестирование** — для проверки комбинаций параметров `recalculate`, `owner` и `region` в связи с их независимостью
- **Предугадывание ошибок** — для выявления потенциальных проблем
